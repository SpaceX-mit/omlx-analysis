# oMLX 分析文档库

本目录存放对 oMLX 项目及其相关技术的深度分析文档。

## 命名与编号规范

所有分析文档遵循统一命名格式：

```
<编号>-<主题>-<子主题>.md
```

- **编号**：三位数字（前导补零），如 `001`、`002`、`023`、`100`
  - 数字大小反映主题层次或时间顺序，**不强制连续**（允许跳号以表达分类）
  - 同一主题相关文档使用连续编号
- **主题**：单词简短描述主题领域（小写，连字符分隔），例如：
  - `omlx` —— oMLX 项目本身的分析
  - `mlx` —— Apple MLX 框架相关
  - `vllm` —— vLLM 系统设计参考
  - `metal` —— Metal GPU 编程
  - `arch` —— 通用架构分析
  - `perf` —— 性能优化
- **子主题**：可选，进一步细化

## 文档模板

每个分析文档建议包含以下章节（可按需裁剪）：

1. **背景与目标** —— 为什么分析这个主题
2. **核心发现** —— 关键结论摘要
3. **详细分析** —— 深入展开
4. **参考与引用** —— 链接、文档、commit hash
5. **附录** —— 可选：图表、时序、代码片段

## 贡献流程

1. 选定下一个可用编号（按上述区间约定）
2. 复制上一个相似主题文档作为模板
3. 写完后在本 README 的「文档清单」表格中追加一行
4. 提交 commit，commit message 建议格式：`docs(analysis): add <编号> <简短标题>`
